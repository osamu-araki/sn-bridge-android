Chrome Bridge Android

Server & Page Fetcher — アーキテクチャ図解・画面説明

概要

Chrome Bridge Android は、macOS 版 Chrome Bridge（Electron + Chrome 拡張）を Android に移植したアプリです。Chrome 拡張が使えない Android 環境では WebView でページ取得を行い、サーバー（Agent）もアプリに内蔵しています。

外部クライアント（n8n / GAS / curl）から HTTP でページ取得を依頼し、Cloudflare Tunnel 経由でインターネットからもアクセスできます。

Tunnel の仕組み（技術詳細）

cloudflared の同梱方法

Go + Android NDK（CGO_ENABLED=1）でビルドした cloudflared ARM64 バイナリを jniLibs/arm64-v8a/libcloudflared.so として APK に同梱。AGP が nativeLibraryDir に展開し、SELinux の execute が許可された領域に配置されます。

起動オプション

  • --token <TOKEN> run: ダッシュボードで作成したトンネルにトークン方式で接続
  • --protocol http2: QUIC を回避（SELinux の netlink_route_socket 制限対策）
  • --no-autoupdate: 自動更新を無効化
  • HOME=filesDir: cloudflared の作業ディレクトリをアプリ領域に設定

DNS 対策（CGO ビルド）

Go のデフォルト DNS リゾルバは /etc/resolv.conf を参照しますが、Android には存在せず、フォールバック先の [::1]:53 / 127.0.0.1:53 もポート 53 が特権ポート（ip_unprivileged_port_start=1024）のため非 root では使えません。

解決策: CGO_ENABLED=1 + Android NDK clang でクロスコンパイルすると、Go が C ライブラリの getaddrinfo() を使うようになり、Android のシステム DNS リゾルバ経由で名前解決が行われます。

ビルドコマンド: CGO_ENABLED=1 GOOS=android GOARCH=arm64 CC=aarch64-linux-android26-clang go build

Cloudflare Tunnel セットアップ

Cloudflare Zero Trust ダッシュボードからトンネルを作成する。ダッシュボードで作成したトンネルはトークン方式で管理され、アプリにトークンを入力するだけで接続できる。

※ Cloudflare Tunnel は無料プラン（Free）で利用可能。帯域・リクエスト数の制限なし。

1. トンネルの作成

  • Cloudflare Zero Trust にログイン
  • Networks → Connectors → Create a tunnel
  • Cloudflared を選択 → トンネル名を入力（例: chrome-bridge）
  • 表示されるトークン（eyJ... 形式）をコピー

2. Public Hostname の設定

  • トンネル作成の続き、または既存トンネルの Configure → Public Hostname タブ
  • Add a public hostname をクリック
  • Subdomain: 任意（例: bridge）、Domain: 管理しているドメインを選択
  • Type: HTTP、URL: localhost:3000
  • Save hostname で保存

3. アプリでの接続

  • アプリの Tunnel Token にトークンを入力
  • Tunnel Domain にサブドメイン（例: bridge.example.com）を入力
  • 「接続」ボタンをタップ → Cloudflare Edge に接続

注意: CLI（cloudflared tunnel create）で作成したトンネルは「ローカル管理型」となり、ダッシュボードから Public Hostname を設定できません。アプリのトークン方式で利用するにはダッシュボードから作成してください。

API 仕様（macOS 版と完全互換）

エンドポイント メソッド 説明
/status GET サーバー状態を返す
{ webview_ready, pending_requests, uptime_seconds }
/fetch POST ページ取得を実行
リクエスト: { url, wait, max_length, timeout, mode }
レスポンス: { ok, mode, url, final_url, title, text, length, elapsed_ms }

認証

Authorization: Bearer <API_KEY> ヘッダーで認証。API Key 未設定時は localhost からのリクエストのみ許可。

レート制限

IP あたり 20 リクエスト / 分。超過時は HTTP 500 を返します。
