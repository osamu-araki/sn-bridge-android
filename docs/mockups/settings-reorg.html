3 パターン比較

パターン 向いている状況 注意点
A. シンプル見出し 最少ノイズ。文字情報が主役で、カテゴリ数が今後変動しても破綻しにくい パッと見の区別力が弱い。スクロールで境目を見落としやすい
B. アイコン付き 視覚的に即区別できる。トラブル時に該当カテゴリへ素早く辿り着ける アイコンの意味を覚える必要。カテゴリ追加のたびに適切なアイコンを選ぶ手間
C. カラードット 最小コストで視覚的なグルーピング感。色だけ覚えれば済む 色弱への配慮。色だけに頼らずテキストも併記しているので実害は小

保存バー（3 パターン共通）

  • 常時画面下に固定（端末画面の高さに関わらず sticky）
  • 変更なし → グレー「変更なし」
  • 変更あり → 橙色ハイライト「● 未保存の変更が N 件」+「破棄」「保存」
  • 保存後、再起動が必要な変更（Port / API Key / Tunnel / 同時並列数）を含む場合のみポップアップで確認

推奨

B. アイコン付きが運用観点で最も実用的です。設定タブを開く時点で「何を直したいか」を持っている運用者にとって、アイコンによる即座のスキャン性が効きます。SalesNow ブランドの teal を保ちつつ、lucide ラインアイコンで装飾過剰にならない範囲に収めています。

シンプルさ重視なら A、軽い差別化なら C も成立します。決め切れなければ B → A の順での導入が無難です。
